A step-by-step guide for starting projects and saving work.

Phase 1: Identity Setup

Goal: Sign your work so GitHub knows it is you. git config --global --replace-all user.name "GarethJ" git config --global --replace-all user.email "jjones@dopainium.com"

Phase 2: The "Save" Cycle (Branching)

1. Create a "Draft" Branch: Never work on main directly. git checkout -b branch-name-here
2. Stage Changes: Put your new work in the "box." git add .
3. Commit: Seal the "box" with a label. git commit -m "Describe what you did here"
4. Push: Send the box to GitHub. git push --set-upstream origin branch-name-here

Phase 3: Merging & Syncing

1. Pull Request: On GitHub.com, click "Compare & pull request" and then "Merge."
2. Back to Main: Switch back to the official version on your Mac. git checkout main
3. Pull: Download the merged work to your Mac. git pull origin main

Antigravity Specifics

Folder Structure: Always put skills here: .antigravity/skills/your-skill.md
Security: If Terminal asks for a password, use your Personal Access Token (PAT), not your GitHub password.
